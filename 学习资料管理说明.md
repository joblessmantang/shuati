# 学习资料管理说明

## 功能概述

学习资料模块提供前端学习书籍与资料的浏览、详情查看、下载及管理功能。

## 数据库

### 表结构

**resources** — 资料表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| title | VARCHAR(200) | 资料标题 |
| author | VARCHAR(100) | 作者 |
| description | TEXT | 简介 |
| category_id | INT | 关联分类ID |
| cover_url | VARCHAR(500) | 封面图URL |
| file_url | VARCHAR(500) | 文件下载地址 |
| file_type | VARCHAR(20) | 文件格式（默认 pdf） |
| file_size | INT | 文件大小（字节） |
| is_recommended | TINYINT(1) | 是否推荐 |
| uploader_id | INT | 上传者ID |
| view_count | INT | 浏览次数 |
| download_count | INT | 下载次数 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

**resource_topics** — 资料与知识点关联表（多对多）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| resource_id | INT | 资料ID |
| category_id | INT | 关联的分类/知识点 |
| created_at | DATETIME | 创建时间 |

### 迁移

运行以下命令创建表结构和示例数据：

```bash
cd houduan
node src/scripts/runMigrate.js
```

执行成功后会输出：`执行成功！资源表、示例数据已就绪。`

### 示例数据

迁移脚本包含 6 条示例资料（ID 1-6），涉及 JavaScript、CSS、Vue、HTTP、HTML 等分类。

## 后端 API

### 用户侧

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/resources | 资料列表（分页、分类筛选、关键词搜索） |
| GET | /api/resources/recommended | 基于用户薄弱点推荐资料 |
| GET | /api/resources/:id | 资料详情 |
| GET | /api/resources/:id/download | 下载资料文件 |

### 管理侧

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| GET | /api/resources/admin/list | 全部资料列表 | 管理员 |
| POST | /api/resources | 新增资料 | 管理员 |
| PATCH | /api/resources/:id | 编辑资料 | 管理员 |
| DELETE | /api/resources/:id | 删除资料 | 管理员 |

### 文件上传

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/upload/cover | 上传封面图（jpg/png/webp，最大 5MB） |
| POST | /api/upload/file | 上传资料文件（PDF，最大 100MB） |

## 前端页面

### 用户页面

- **资料库列表** — `shuati/src/views/Resources.vue`
  - 路径：`/resources`（hash 模式：`/#/resources`）
  - 功能：资料列表（分类筛选、关键词搜索、分页）、推荐区、详情弹窗、下载按钮

- **详情弹窗** — `shuati/src/components/ResourceDetail.vue`
  - 功能：封面、标题、作者、简介、适合的知识点标签、下载按钮

### 管理页面

- **资料管理** — `shuati/src/views/admin/AdminResources.vue`
  - 路径：`/admin/resources`（hash 模式：`/#/admin/resources`）
  - 功能：资料列表、新增资料（含封面和文件上传）、编辑资料、删除资料、标记推荐

## 下载机制

点击下载按钮时，前端通过 `window.open` 打开后端下载接口：

```
http://localhost:3000/api/resources/{id}/download
```

后端处理流程：
1. 根据 ID 查询数据库中的 `file_url`
2. 解析文件路径（相对于 `houduan/uploads/files/`）
3. 检查文件是否存在
4. 通过 `res.download()` 返回文件，或返回 404

## 文件存储

- 封面图：`houduan/uploads/covers/cover_{timestamp}.{ext}`
- 资料文件：`houduan/uploads/files/file_{timestamp}.{ext}`
- 静态服务：`GET /uploads/*` 由 Express 直接提供

## 注意事项

1. **迁移必须运行**：新增资料功能前需先执行 `node src/scripts/runMigrate.js` 创建表
2. **生产环境**：前端 `baseUrl` 需修改为实际后端地址
3. **权限控制**：管理侧 API 通过 `authMiddleware` 中间件校验登录状态和角色
4. **文件路径**：下载时后端通过拼接 `__dirname + '../../' + file_url` 定位文件，确保文件在 `uploads/files/` 目录下
