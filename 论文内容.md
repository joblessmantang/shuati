# 智能刷题平台的设计与实现



## 摘要

随着前端技术的快速发展和前端面试难度的不断提升，开发者需要一个高效、专业的刷题辅助工具来提升面试准备效率。传统刷题方式存在题目分散、答案不详细、无法针对性训练等痛点，而大语言模型技术的成熟为解决这些问题提供了新的可能性。

本设计实现了一个面向前端面试备考的智能刷题平台，采用前后端分离架构，前端基于 Vue 3 构建，后端基于 Node.js + Express + MySQL。平台提供题目练习、AI 智能提示、论坛讨论等核心功能，支持多种练习模式、错题自动归集、AI 追问详解等特性。

在刷题练习方面，平台实现了三种练习模式：全部题目模式支持从题库随机抽取指定数量的题目进行练习；仅错题模式允许用户仅针对历史错题进行强化训练；混合模式则优先包含错题，剩余名额由其他题目填充，满足用户多样化的练习需求。答题过程中，系统提供实时计时、暂停计时、答题卡跳转、收藏题目等辅助功能。交卷后自动记录错题和练习历史，帮助用户量化学习效果。

在 AI 智能辅助方面，平台通过精心设计的提示词工程实现“授人以渔”式的学习引导。答题过程中，用户点击“AI 提示”按钮，系统结合题目内容和用户已选答案返回引导性提示，不直接输出答案，旨在帮助用户独立思考找到正确答案。交卷后，“查看详解”按钮激活，用户可获取题目的详细解释。更重要的是，用户可在文本框中输入具体问题（如“为什么选 C 不对？”），AI 将结合用户输入生成个性化追问解答，实现深度学习。

在社区交流方面，论坛模块允许用户发布帖子（可关联具体题目，也可不关联）、浏览帖子列表（支持关键词搜索）、查看帖子详情及评论、发表评论，形成知识共享的学习社区。帖子浏览量在访问详情时自动统计，关联题目的帖子使用 LEFT JOIN 兼容无关联题目的场景。

本论文从系统分析、需求分析、业务流程、架构设计、数据库设计、接口设计、安全设计、系统实现及测试等方面详细阐述该系统的设计与实现过程。系统经过功能测试和安全测试，核心功能全部正常运行，能够有效帮助前端开发者提升面试准备效率。

**关键词**：智能刷题平台；Vue 3；Node.js；MySQL；AI 提示词工程；前后端分离



## Abstract

With the rapid development of front-end technology and the increasing difficulty of front-end interviews, developers need an efficient and professional tool to improve their interview preparation efficiency. Traditional practice methods suffer from scattered questions, incomplete answers, and the inability to provide targeted training. The maturity of large language model technology provides new possibilities for solving these problems.

This design implements an intelligent practice platform for front-end interview preparation, adopting a front-end and back-end separation architecture. The front-end is built based on Vue 3, and the back-end is based on Node.js + Express + MySQL. The platform provides core functions such as question practice, AI intelligent hints, and forum discussions, supporting multiple practice modes, automatic error question collection, and AI follow-up question explanations.

In terms of question practice, the platform implements three practice modes: the all-questions mode supports randomly selecting a specified number of questions from the question bank for practice; the error-only mode allows users to focus on historical error questions for reinforcement training; the mixed mode prioritizes error questions, with remaining quotas filled by other questions, meeting diverse practice needs. During answering, the system provides real-time timing, pause timing, answer sheet navigation, and favorite question features. After submission, error questions and practice history are automatically recorded, helping users quantify learning effects.

In terms of AI intelligent assistance, the platform achieves a "teaching people to fish" style of learning guidance through carefully designed prompt engineering. During answering, when users click the "AI Hint" button, the system combines question content and the user's selected answer to return guiding hints without directly outputting answers, aiming to help users independently find the correct answer. After submission, the "View Details" button is activated, allowing users to obtain detailed explanations of questions. More importantly, users can input specific questions in the text box (such as "Why is choosing C wrong?"), and AI will generate personalized follow-up answers based on user input, achieving deep learning.

In terms of community communication, the forum module allows users to post (which can be associated with specific questions or not), browse post lists (supporting keyword search), view post details and comments, and post comments, forming a knowledge-sharing learning community. Post view counts are automatically recorded when viewing details, and posts associated with questions use LEFT JOIN to be compatible with posts without associated questions.

This paper elaborates on the system's design and implementation process from aspects of system analysis, requirement analysis, business process, architecture design, database design, interface design, security design, system implementation, and testing. The system has undergone functional testing and security testing, with all core functions operating normally, and can effectively help front-end developers improve interview preparation efficiency.

**Keywords**: Intelligent Practice Platform; Vue 3; Node.js; MySQL; AI Prompt Engineering; Front-end and Back-end Separation

## 第1章 绪论

### 1.1 研究背景与意义

在互联网行业快速发展的今天，前端技术日新月异，JavaScript、CSS、HTML 等核心知识的考察始终是技术面试的重点环节。随着 Web 应用的复杂度不断提升，企业对前端开发者的要求也越来越高，不仅要求掌握基础知识，还需要深入理解框架原理、性能优化、异步编程等高级主题。面试题目从最初的基础语法考察，逐渐演变为对综合能力的全面评估，包括代码理解能力、问题分析能力、性能意识等多个维度。

面对日益增长的面试难度，传统的刷题方式暴露出诸多痛点。首先，题目资源分散，用户需要在多个平台之间切换，难以形成系统化的练习体系。其次，现有题库的答案往往过于简略，仅给出正确答案而缺乏详细解释，用户知其然不知其所以然。再次，传统刷题工具缺乏针对性，无法根据用户的历史表现智能推荐需要强化的知识点。最后，刷题过程缺乏互动性和趣味性，用户容易产生倦怠感，学习效率低下。

随着大语言模型（Large Language Model，LLM）技术的快速发展，以 GPT、DeepSeek 为代表的 AI 模型在自然语言处理、代码理解、知识问答等领域展现出惊人的能力。将 AI 能力融入学习辅助工具，能够为用户提供个性化的学习指导，显著提升学习效率。AI 不仅能够提供详细的题目解析，还能够根据用户的具体问题生成针对性的回答，实现一对一的辅导效果。

基于以上背景，本项目设计并实现了一个面向前端面试备考的智能刷题平台。本项目的意义主要体现在以下几个方面：

第一，资源整合。本平台汇聚高质量前端面试题目，涵盖 JavaScript、CSS、HTML、Vue 框架、网络协议等多个分类，每道题目都经过精心筛选和设计，确保质量和实用性。平台提供系统化的练习环境，帮助用户高效备考。

第二，AI 赋能。本系统通过提示词工程实现“授人以渔”式的学习引导，而非直接给出答案。答题过程中，AI 提供引导性提示帮助用户独立思考；交卷后，用户可以追问具体问题获得个性化解答。这种方式能够真正提升用户的知识掌握程度，而非简单的记忆答案。

第三，社区互动。论坛模块允许用户就特定题目展开讨论，分享学习心得和解题思路。用户可以发布与题目相关的讨论帖，也可以浏览其他用户的经验分享，形成知识共享的学习社区。这种互动不仅能够帮助用户解决具体问题，还能够拓展思路、加深理解。

第四，数据驱动。通过错题本、练习历史等数据，系统帮助用户量化学习效果。用户可以清晰地看到自己的正确率变化、薄弱知识点分布、练习时长趋势等信息，从而有针对性地调整学习计划。这种数据驱动的方式使得学习更加科学、高效。

### 1.2 研究目标

本项目旨在设计并实现一个功能完整的前端面试智能刷题平台，具体目标包括以下几个方面：

在系统架构方面，构建稳定可靠的前后端分离 Web 应用。前端采用 Vue 3 框架，利用其强大的响应式系统和组件化开发能力，提供流畅的用户体验。后端采用 Node.js + Express 架构，利用其非阻塞 I/O 特性处理高并发请求。数据库采用 MySQL，确保数据的持久化和可靠性。前后端通过 RESTful API 进行通信，实现职责分离，便于独立开发和部署。

在刷题功能方面，实现多种练习模式以满足不同用户的需求。全部题目模式从题库中随机抽取指定数量的题目进行练习；仅错题模式允许用户仅针对历史错题进行强化训练；混合模式则优先包含错题，剩余名额由其他题目填充。答题过程中支持实时计时、暂停计时、答题卡跳转、收藏题目等辅助功能。交卷后自动记录错题和练习历史，自动禁用相关按钮防止二次提交。

在 AI 功能方面，集成智能提示和追问详解功能。答题过程中，用户点击“AI 提示”按钮，系统结合题目内容和用户已选答案返回引导性提示，严格控制不直接输出答案。交卷后，“查看详解”按钮激活，用户可获取题目的详细解释和知识点分析。更重要的是，系统支持用户自由追问，用户可以针对具体选项或知识点提出问题，AI 将生成针对性的个性化解答。

在社区功能方面，提供论坛交流模块，支持帖子发布与评论互动。用户可以发布与具体题目相关的讨论帖，也可以发布一般性的技术讨论。帖子支持关键词搜索，便于快速找到感兴趣的内容。浏览帖子时自动统计浏览量，评论功能支持用户之间的互动交流。

在安全机制方面，设计完善的安全机制保障用户数据安全。用户密码使用 bcrypt 算法加密存储，JWT Token 实现无状态认证，SQL 注入防护使用参数化查询，XSS 攻击防护对用户输入进行转义处理，题目答案仅在后端验证不暴露给前端。

## 第2章 相关技术介绍

### 2.1 前端技术栈

本系统前端采用 Vue 3 作为核心框架，配合 Vite 构建工具、Vue Router 路由管理、Pinia 状态管理、Axios HTTP 客户端等技术构建现代化单页应用。

Vue 3 是目前最主流的前端框架之一，相比 Vue 2 进行了全面升级。Vue 3 引入了全新的 Composition API，配合 `<script setup>` 语法糖，使得代码更加简洁和组织良好。Composition API 提供了 `ref`、`reactive`、`computed`、`watch`、`watchEffect` 等响应式 API，开发者可以更加灵活地组织代码逻辑，将相关功能的代码集中在一起，而非按照选项类型分散在 `data`、`methods`、`computed` 等不同区域。这种组织方式在处理复杂组件时优势明显，也更利于代码复用和单元测试。Vue 3 还对虚拟 DOM 进行了重构，编译时优化使得运行时的性能得到显著提升。Proxy 响应式系统替代了 Vue 2 的 Object.defineProperty，解决了无法检测对象属性添加删除的问题。

Vite 是新一代的前端构建工具，由 Vue.js 作者尤雨溪主导开发。Vite 利用浏览器原生 ES Module 支持实现极快的冷启动速度，无需等待打包过程即可启动开发服务器。热模块替换（HMR）也得到了极大优化，修改代码后可以在几十毫秒内看到更新结果。Vite 还支持 Vue 3、TypeScript、CSS 预处理器等常用技术，生产环境使用 Rollup 进行优化打包。相比 Webpack，Vite 的配置更加简洁，开发体验更加流畅。

Vue Router 4 是 Vue.js 的官方路由管理库，用于构建单页应用（SPA）的路由系统。Vue Router 4 支持嵌套路由、路由参数、路由守卫等功能。本系统使用路由守卫实现页面访问控制，未登录用户访问需要认证的页面时自动跳转到登录页。路由守卫还可以用于加载进度条、页面标题更新等场景。

Pinia 是 Vue 3 的新一代状态管理方案，官方推荐使用。Pinia 相比 Vuex 更加轻量简洁，API 设计更加直观。Pinia 支持模块化组织状态，提供 `storeToRefs` 工具函数方便在模板中使用响应式状态，devtools 集成良好。本系统使用 Pinia 管理用户登录状态，实现全局共享的用户信息。

Axios 是前端最常用的 HTTP 请求库，用于与后端 API 通信。本系统对 Axios 进行统一封装，配置请求拦截器自动注入 JWT Token，配置响应拦截器统一处理错误响应和 Token 过期跳转。统一的 API 层封装使得业务代码更加简洁，也便于统一管理接口地址和错误处理。

SweetAlert2 是一个美观、响应式的弹窗库，用于替代浏览器原生的 `alert`、`confirm` 等对话框。SweetAlert2 提供丰富的配置选项，可以自定义弹窗标题、内容、图标、按钮等，支持动画效果和回调函数。本系统使用 SweetAlert2 实现确认交卷、删除确认、操作成功提示等功能。

样式方案采用 CSS Variables 实现主题变量管理，通过定义统一的颜色、字体、间距等变量，方便实现主题切换和样式一致性。组件样式使用 Scoped CSS 隔离，避免样式冲突。

### 2.2 后端技术栈

本系统后端采用 Node.js 作为运行环境，Express.js 作为 Web 框架，MySQL 作为数据库，JWT 用于用户认证，bcryptjs 用于密码加密。

Node.js 是基于 Chrome V8 引擎的 JavaScript 运行时环境，允许开发者使用 JavaScript 开发服务端应用。Node.js 的核心特性是非阻塞 I/O 和事件驱动，能够高效处理大量并发连接，非常适合 I/O 密集型的 Web 应用。Node.js 还拥有庞大的 npm 包生态，几乎所有前端开发所需的功能都有成熟的第三方库支持。JavaScript 全栈统一意味着开发者可以使用相同的语言和思维方式开发前端和后端，降低了学习成本。

Express.js 是基于 Node.js 的轻量级 Web 应用框架，采用中间件模式组织代码。Express 的设计哲学是保持核心简单，通过中间件扩展功能。常见的中间件包括路由、静态文件服务、CORS 跨域处理、请求体解析、日志记录等。Express 的路由系统支持 HTTP 方法、路径参数、查询参数、正则匹配等功能，开发 RESTful API 非常方便。相比 Koa、Hapi 等框架，Express 更加成熟稳定，社区资源丰富。

MySQL 是应用最广泛的开源关系型数据库，本系统使用 MySQL 8.0 版本。MySQL 支持标准 SQL 语法，具备高性能、高可靠性、丰富的数据类型支持等优点。本系统使用 mysql2 库的 promise 版本，支持异步查询和连接池管理。连接池技术复用数据库连接，避免频繁创建销毁连接带来的性能开销，提高系统的并发处理能力。

JWT（JSON Web Token）是一种开放标准（RFC 7519），用于在各方之间安全地传输信息。JWT 由三部分组成：Header（头部）、Payload（负载）和 Signature（签名）。Header 部分指定签名算法，Payload 部分包含声明信息（如用户 ID、过期时间等），Signature 部分是使用密钥对前两部分的签名。JWT 是无状态的，服务器不需要存储 Token 信息，适合分布式系统和微服务架构。本系统使用 jsonwebtoken 库签发和验证 Token。

bcryptjs 是 bcrypt 算法的 JavaScript 实现，用于安全地哈希密码。bcrypt 采用自适应哈希算法，可以配置 cost factor（计算强度），随着硬件性能提升可以增加计算成本，抵御暴力破解。bcrypt 算法内置盐值（salt），每次哈希结果都不同，防止彩虹表攻击。密码哈希是单向的，无法从哈希值反推原始密码，登录时通过 compare 方法比较输入密码与存储哈希是否匹配。

AI 能力通过调用外部大模型 API 实现，本系统使用 DeepSeek 作为 AI 服务提供商。DeepSeek 提供了 GPT 兼容的 API 接口，支持流式输出和多轮对话。后端统一封装 AI 调用，避免前端暴露 API Key，提高安全性。Axios 库用于发起 HTTP 请求调用 AI 接口。

dotenv 是 Node.js 的环境变量管理库，允许将敏感配置（如数据库密码、API Key 等）存储在 .env 文件中，与代码分离。生产环境和开发环境可以使用不同的配置文件，便于部署和管理。

### 2.3 技术架构选型依据

本系统选择 Vue 3 + Node.js 的前后端分离架构，基于以下几方面的综合考虑：

在开发效率方面，Vue 3 的组件化开发模式使得前端代码易于维护和复用，Composition API 使得复杂逻辑的组织更加清晰。Node.js 使用 JavaScript 作为开发语言，前端开发者可以零成本上手，实现全栈开发。Express 框架简洁灵活，中间件机制使得功能扩展非常方便。npm 生态拥有海量的第三方库，几乎所有常见需求都有现成的解决方案。

在性能表现方面，Node.js 的非阻塞 I/O 模型非常适合 Web 应用场景。Web 应用的主要瓶颈在于 I/O 操作（数据库查询、文件读写、网络请求），而非 CPU 计算。Node.js 在等待 I/O 完成时可以处理其他请求，充分利用系统资源。对于本系统的实际负载（用户量有限，以 CRUD 操作和 AI 调用为主），Node.js 完全能够胜任。

## 第4章 系统设计

### 4.1 总体架构设计

#### 4.1.1 系统架构

本系统采用经典的前后端分离架构（Front-End / Back-End Separation Architecture），前端与后端通过 JSON over HTTP 通信，各自独立开发、测试和部署。这种架构的核心思想是将表示层和业务逻辑层分离，前端负责用户界面和交互逻辑，后端负责业务处理和数据存储。前后端通过预定义的 API 接口进行通信，接口遵循 RESTful 规范，使用 JSON 格式传输数据。

前端采用 Vue 3 构建单页应用（SPA）。单页应用的核心特点是在首次加载时下载完整的应用代码，之后的页面切换不需要重新加载整个页面，而是通过 JavaScript 动态更新 DOM 实现。这种方式提供了流畅的用户体验，页面切换几乎没有延迟。Vue Router 负责管理页面路由，支持路由参数、嵌套路由、路由守卫等功能。Pinia 负责管理全局状态，包括用户登录信息、应用配置等。Axios 封装了 HTTP 请求，配置了请求拦截器和响应拦截器，实现 Token 自动注入、错误统一处理、401 自动跳转登录页等功能。

后端采用 Node.js + Express 构建 RESTful API 服务。Express 框架使用中间件模式组织代码，请求经过一系列中间件处理后到达路由处理器。路由处理器调用 Controller 层，Controller 层处理请求参数、调用 Service 层、返回响应。Service 层封装业务逻辑，可能需要访问数据库。数据库使用 MySQL，通过 mysql2 的连接池进行管理。

AI 功能由后端统一封装。后端调用外部大模型 API（如 DeepSeek），将结果返回给前端。这种设计有多个好处：第一，避免前端暴露 API Key，提高安全性；第二，便于统一处理 AI 相关的错误和重试逻辑；第三，如果需要更换 AI 服务提供商，只需修改后端代码，前端无需改动。

安全方面，用户密码使用 bcryptjs 加密存储，即使数据库泄露也无法获取原始密码。JWT Token 实现无状态认证，Token 包含用户 ID 和过期时间，后端验证 Token 即可确认用户身份。API 路由通过 authMiddleware 中间件保护，未认证请求返回 401。SQL 注入防护使用参数化查询，用户输入不参与 SQL 语句的拼接。XSS 防护对论坛内容进行 HTML 特殊字符转义。

#### 4.1.2 部署架构

开发环境下，前端通过 Vite 开发服务器（端口 5173）运行。Vite 配置了 proxy 将 `/api` 请求代理到后端 Express 服务器（端口 3000），避免跨域问题。开发者需要同时运行前端和后端两个服务，前端开发时修改代码会自动热更新，后端开发时使用 nodemon 监听文件变化自动重启。

生产环境下，前端通过 `npm run build` 命令打包为静态资源文件（包括 HTML、CSS、JavaScript、图片等）。这些静态文件可以由后端 Express 直接托管，也可以部署到 Nginx 等 Web 服务器。当后端配置 `NODE_ENV=production` 时，Express 会托管前端打包文件，所有非 API 请求返回 index.html，支持 SPA 的路由机制。也可以将前端打包文件部署到 CDN 或对象存储，通过 Nginx 反向代理到后端 API，实现动静分离。

这种前后端分离的部署方式灵活性很高：小型项目可以简化部署，后端直接托管前端；中大型项目可以分离部署，前端使用 CDN 加速，后端使用负载均衡提高可用性；微服务架构下，多个后端服务可以独立部署和扩展。

### 4.2 功能模块划分

系统按功能划分为以下核心模块：

用户认证模块负责用户注册、登录、Token 签发、角色管理等功能。用户注册时验证用户名唯一性，密码使用 bcrypt 加密后存储。登录时验证用户名密码，签发 JWT Token。管理员可以查看用户列表、修改用户角色、删除用户。关键文件包括 `authController.js` 和 `auth.js`（中间件）。

题目管理模块负责题目的 CRUD 操作、分类筛选、关键词搜索等功能。题目数据存储在 questions 表，包含题干、代码片段、选项、正确答案、分类等信息。答案字段仅在后端验证时使用，不暴露给前端。关键文件包括 `questionController.js` 和 `Quiz.vue`。

AI 智能助手模块是本系统的核心创新点，负责 AI 提示和追问详解功能。getHint 方法在答题中使用，根据题目和用户已选答案返回引导性提示，不直接给答案。getExplanation 方法在交卷后使用，返回详细解释，支持用户自由追问。提示词工程严格控制 AI 输出，确保不给答案只给思路。系统实现了结果缓存，30 分钟内重复请求同一题目的提示直接返回缓存结果。关键文件包括 `aiController.js`、`aiService.js` 和 `AiHint.vue`。

论坛交流模块负责帖子发布、评论互动、关联题目讨论等功能。帖子可以关联具体题目，也可以不关联。帖子浏览量在访问详情时自动统计。评论功能支持用户之间的互动。系统实现了关键词搜索，支持帖子标题、内容及关联题目标题的三字段模糊匹配。关键文件包括 `postController.js`、`commentController.js` 和 `Forum.vue`。

错题本模块负责自动记录用户答错的题目。交卷后系统自动将答错的题目写入错题本，无需用户手动操作。错题本按用户筛选，用户只能查看自己的错题。同一个用户同一道题目只记录一次，防止重复。关键文件包括 `wrongBookController.js` 和 `WrongBook.vue`。

收藏模块允许用户主动收藏感兴趣的题目。与错题本不同，收藏是用户主动行为，可以在答题过程中随时操作。收藏的题目便于后续复习。关键文件包括 `favoritesController.js`。

练习历史模块负责记录每次练习的成绩、用时、时间戳。练习历史数据用于生成统计图表，帮助用户了解学习趋势。关键文件包括 `historyController.js` 和 `History.vue`。

数据分析模块负责练习数据的可视化展示。通过 ECharts 图表展示练习趋势、正确率变化、薄弱知识点分布等信息。关键文件包括 `Analytics.vue`。

分类管理模块负责题目分类的增删改查。初始数据包含 5 大分类（JavaScript、CSS、HTML、Vue/前端框架、HTTP/网络）和 100 道精选题目。关键文件包括 `categoryController.js`。

举报管理模块允许用户举报题目问题（题目错误、选项错误等），管理员可以查看和处理举报。关键文件包括 `reportController.js` 和 `AdminReports.vue`。

中间件模块提供通用的中间件功能。authMiddleware 验证 JWT Token，保护需要认证的路由。errorHandlerMiddleware 统一处理错误，格式化错误响应。关键文件包括 `auth.js` 和 `errorHandler.js`。

### 5.2 前端实现

#### 5.2.1 API 封装层

前端通过 Axios 封装统一的 HTTP 请求层，简化业务代码中的 API 调用。request 实例配置了 baseURL 为环境变量或默认值，设置了 Content-Type 头。请求拦截器自动从 localStorage 读取 Token，注入到 Authorization 头中。响应拦截器统一处理错误，对于 401 错误清除本地存储并跳转登录页，对于其他错误统一抛出，方便调用方 catch。

API 模块按照功能划分，authApi、questionApi、aiApi、postApi、commentApi、wrongBookApi、favoritesApi、historyApi、categoryApi、reportApi 分别封装对应模块的接口调用。这种封装方式使得接口地址集中管理，便于维护和修改。

#### 5.2.2 答题页面核心实现

Quiz.vue 是整个平台的核心页面，负责刷题流程的完整管理。页面使用状态机模式管理答题状态：not_started（未开始）、in_progress（进行中）、submitted（已交卷）。

组件在挂载时加载所有题目、分类、用户错题、用户收藏等数据。handleStartQuiz 方法根据用户选择的模式和数量，从题库中筛选题目。全部模式直接随机抽取；错题模式从错题本中筛选；混合模式优先包含错题，剩余名额由其他题目填充。选中的题目经过随机打乱后进入答题环节。

计时器使用 setInterval 实现，每秒更新一次显示。暂停功能通过清除定时器实现，恢复时重新启动定时器。交卷时需要用户确认，确认后禁用交卷和暂停按钮，防止重复提交。

提交逻辑首先逐题比对用户答案与正确答案，计算得分。然后对答错的题目调用错题本 API 记录。最后调用历史记录 API 保存本次练习的成绩和用时。操作完成后显示成绩统计界面。

#### 5.2.3 AI 提示组件实现

AiHint.vue 组件实现了 AI 辅助学习的核心功能。组件接收当前题目、用户答案、是否交卷等 props，根据状态渲染不同的交互界面。

答题中状态下，用户点击"AI 提示"按钮调用 getHint 方法。请求参数包含题目 ID、题干、代码片段、选项、用户已选答案。AI 结合这些信息返回引导性提示，但不给出答案。

交卷后状态下，用户点击"查看详解"按钮调用 getExplanation 方法。此时请求参数额外包含正确答案，AI 可以给出完整解释。用户还可以在文本框中输入追问问题（如"为什么选 C 不对？"），AI 结合用户问题返回针对性解答。

组件使用 v-model 绑定提示内容、加载状态、错误信息等响应式数据。错误处理显示友好的错误提示，而非技术性错误信息。

#### 5.2.4 论坛页面实现

Forum.vue 实现了帖子浏览、搜索、发布和评论功能。页面包含帖子列表和帖子详情两个视图，通过状态切换。

帖子列表加载时调用 postApi.list 接口，支持关键词搜索参数。返回数据包含 posts 数组和 pagination 对象，前端正确提取嵌套字段进行渲染。

查看帖子详情时调用 postApi.get 接口，返回包含帖子信息和评论列表的嵌套数据。浏览量在请求后端时自动增加。

发布帖子功能通过弹窗表单实现，包含标题、内容、关联题目（可选）等字段。关联题目支持搜索选择，也可以不关联。

发表评论功能在帖子详情下方，输入评论内容后提交。后端返回新评论数据，前端乐观更新评论列表和评论数。

### 5.3 后端实现

#### 5.3.1 AI 服务实现

AI 服务是本系统的核心创新点之一。aiService.js 封装了 AI 调用的所有逻辑，包括提示词工程、API 调用、结果缓存等。

getHintPrompt 方法组装答题中使用的提示词。提示词包含角色设定（友好的面试刷题助手）、题目内容、选项列表、用户当前答案。然后给出明确的约束：不要直接给出答案、只给思路和解题方向、简洁有条理、如果用户有答案分析对错。严格的提示词约束确保 AI 不会直接告诉用户答案，而是引导用户独立思考。

getExplainPrompt 方法组装交卷后使用的提示词。角色设定为耐心的编程老师，提示词包含题目内容、选项列表、正确答案、学生的问题。如果学生没有输入具体问题，使用默认的全面解释要求。提示词鼓励详细解释正确答案、分析相关知识点、进行面试拓展。

callAI 方法封装了外部 AI API 的调用逻辑。使用 axios 发送 POST 请求到配置好的 API 地址，请求体包含模型名称、消息列表、温度参数（0.7 平衡创造性和准确性）、最大令牌数（1000）。配置了 60 秒超时时间，避免请求长时间挂起。API Key 从环境变量读取，不硬编码在代码中。

getHint 方法实现了结果缓存。对于同一道题目的提示请求，首先检查缓存中是否存在有效记录（30 分钟内）。缓存命中则直接返回缓存结果，避免重复调用 AI API，节省成本和提高响应速度。缓存未命中则调用 AI 服务，将结果存入缓存后返回。

getExplanation 方法不进行缓存，因为用户可能针对同一道题目进行多次不同的追问，每次都需要调用 AI 生成新的回答。

#### 5.3.2 JWT 认证中间件

authMiddleware 是保护 API 路由的核心中间件。中间件首先检查请求头中是否存在 Authorization 字段，且格式为"Bearer token"。

使用 jwt.verify 方法验证 Token 的有效性。验证通过后，从 Token 中解析出用户 ID 和用户名。然后从数据库查询用户的完整信息（包括角色），验证用户确实存在。

将用户信息挂载到 req.user 对象上，后续的 Controller 可以直接访问。验证失败根据错误类型返回不同的错误信息：未提供 Token、Token 格式错误、Token 已过期、Token 无效、用户不存在等。

#### 5.3.3 数据库连接池配置

database.js 配置了 mysql2 的连接池。使用 createPool 方法创建连接池，配置项包括主机地址、端口、用户名、密码、数据库名等。

waitForConnections 设置为 true，当连接池已满时等待空闲连接。connectionLimit 设置为 10，限制最大连接数为 10。queueLimit 设置为 0，表示队列无限制，超出的请求会等待。enableKeepAlive 启用连接保活，keepAliveInitialDelay 设置为 0 表示立即发送初始探测包。

连接池复用连接，避免频繁创建销毁连接的性能开销。一个请求完成后释放连接回连接池，供下一个请求使用。

### 5.4 难点与解决方案

#### 5.4.1 前后端 API 响应格式不一致

问题描述：部分 Controller 返回 `{ success, data }` 包装格式，部分 Controller 直接返回数据数组。导致前端 Axios 拦截器解包逻辑不一致，Analytics.vue 和 History.vue 无法正确处理。

解决方案：统一所有 Controller 响应格式为 `{ success: true, data: ... }`。前端拦截器保持现有解包逻辑，对于边界情况使用 Array.isArray() 兜底判断。

#### 5.4.2 mysql2 execute 与 LIMIT/OFFSET 兼容问题

问题描述：pool.execute 使用预编译协议，MySQL 预编译对 LIMIT/OFFSET 参数类型有兼容性问题，导致论坛帖子列表返回 500 错误。

解决方案：将 pool.execute 改为 pool.query，LIMIT/OFFSET 使用 parseInt 验证后的数字字面量拼接，其余参数保持占位符传递。

#### 5.4.3 错题本重复插入

问题描述：使用“先查后插”逻辑，高并发下两个请求可能同时通过存在性检查，导致 UNIQUE 约束冲突。

解决方案：数据库表设置 UNIQUE KEY unique_user_question 作为双重保险。在 catch 中判断 err.code === 'ER_DUP_ENTRY' 并友好提示，而非抛出 500 错误。

## 结论

本论文完整阐述了智能刷题平台从需求分析到系统测试的完整设计开发过程。该平台基于 Vue 3 + Node.js + MySQL 的前后端分离架构，实现了刷题练习、AI 智能提示与追问、论坛交流、错题管理、数据分析等完整功能。

系统的主要创新点包括：

第一，AI 提示词工程。通过精心设计的提示词实现“授人以渔”，在答题中提供引导性提示而非直接给答案。这种方式能够真正提升用户的知识掌握程度，而非简单的记忆答案。提示词工程还约束了 AI 的输出边界，确保不直接泄露答案，保持学习的挑战性。

第二，追问详解机制。交卷后支持用户自由追问，AI 结合用户具体问题生成个性化解答。用户可以针对具体选项或知识点提出问题，获得针对性的辅导。这种多轮对话的方式模拟了一对一辅导的场景，大幅提升了学习效果。

第三，多模式练习。支持全部题目、仅错题、混合三种练习模式，满足用户多样化的练习需求。错题自动归集功能免去了用户手动整理错题的麻烦，使复习更加高效。用户可以根据自己的薄弱环节选择合适的练习模式，实现针对性的强化训练。

第四，安全防护体系。bcrypt 密码加密确保密码安全，JWT 无状态认证便于分布式部署，SQL 注入防护保护数据库安全，XSS 过滤保护用户数据，答案保密机制维护考试公平。这些安全措施共同构成了系统的安全防线，保护用户数据和系统稳定。

系统经过功能测试和安全测试，核心功能全部正常运行，安全测试基本达标。总体而言，该系统能够有效帮助前端开发者提升面试准备效率，具有一定的实用价值和推广意义。

未来可以在以下方面进行改进：引入 Redis 缓存提升 AI 响应速度，增加用户学习计划功能，实现题目的分类难度标记，集成更多的 AI 模型选择，支持移动端原生应用开发等。
